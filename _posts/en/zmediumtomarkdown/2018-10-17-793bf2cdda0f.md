#### Introduction

If you've only heard the term "machine learning" but are unclear about its meaning, here's a simple explanation:

> **"Predicting the outcome of similar future events based on your past experiences."**

> For example: If I always add ketchup to my egg pancake, after a few times, the breakfast shop owner will remember, "Handsome, would you like ketchup?" I reply, "Yes" â€” the owner predicts correctly; if I respond, "No, because it's radish cake + egg pancake" â€” the owner remembers and corrects their understanding the next time the same situation arises.

> Input data: egg pancake, cheese egg pancake, egg pancake + radish cake, radish cake, egg

> Output data: add ketchup / do not add ketchup

> Model: the owner's memory and judgment

#### Getting Started

The first step, of course, is the "model" mentioned earlier. Where does the model come from?

There are three ways:
- Find pre-trained models online and convert them to CoreML format.

The [Awesome-CoreML-Models](https://github.com/likedan/Awesome-CoreML-Models){:target="_blank"} GitHub project collects many pre-trained models.

For model conversion, refer to the [official website](https://developer.apple.com/machine-learning/build-run-models/){:target="_blank"} or online resources.
- Appleâ€™s [Machine Learning website](https://developer.apple.com/machine-learning/build-run-models/){:target="_blank"} has a section at the bottom for Download Core ML Models, where you can download models that Apple has trained for us (mainly for learning or testing purposes).

### Today's Main Focus â€” Text Content Classification + **Training Your Own Model**

In simple terms, we provide the machine with "text content" and "categories" to train the computer to classify future data. For example: "Click to view the latest offers!" or "Get $1000 shopping credit now" => "Advertisement"; "Alan sent you a message" or "Your account is about to expire" => "Important matters."

Practical applications: spam detection, label generation, classification prediction.

#### Preparing Training Data:

![Data structure as shown in the image, supports .json, .csv files](/assets/793bf2cdda0f/1*bqKGHErvqhd6gIKCnvve4Q.png)

The data structure is as shown in the image, supporting .json and .csv files.

Prepare the data you want to use for training; here, we will export the training data using Phpmyadmin (Mysql).
```sql
SELECT `title` AS `text`,`type` AS `label` FROM `posts` WHERE `status` = '1'
```

![Change the export format to JSON](/assets/793bf2cdda0f/1*fc10j10OzmI2TGemaqlDmw.png)

Change the export format to JSON.
```json
[
  {"type":"header","version":"4.7.5","comment":"Export to JSON plugin for PHPMyAdmin"},
  {"type":"database","name":"db"},
  {"type":"table","name":"posts","database":"db","data":
    // Remove the above
    [
      {
         "label":"",
         "text":""
      }
    ]
    // Remove the below
  }
]
```

Open the downloaded JSON file and keep only the content in the middle DATA structure.

**Specific Training Method:**
1. First, segment the input sentences (e.g., I want to know what to prepare for the wedding => I want, to know, wedding, need, prepare, what), and then see what its classification is to perform a series of machine learning calculations.
2. Group the training data, for example: 80% is used for training and the other 20% is used for testing and validation.
