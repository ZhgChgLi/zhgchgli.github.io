#### 技術
- **Blog 架構:** Jekyll 靜態網站
- **文章原始檔:** Markdown 檔案
- **伺服器/網頁寄存:** GitHub Pages \(免費可靠\)
- **Engineering:** Claude Code Max \($100 USD，5 小時額度沒用完就做好了\)
- **Design:** Claude Design \(Blog 頁面不多，2 小時額度內就夠用\)

#### 花費
- Claude Code Max $100 USD
- 伺服器/網頁寄存: $0 USD
- Jekyll Theme: $0 USD
- 時間: 一個週末的下午

![](/assets/6bf79c5b4dab/1*pBQcJF9Bim581CsPHGdlxQ.png)

寬版面 / 標準版面

### 實現小 Tips
- 可以跟 AI 說： `你的 Blog 會部署到 GitHub Pages` or XXX，因為每個平台可能機能不一樣，例如 GitHub Pages 只允許部分的 [Jekyll Plugins](https://github.com/github/pages-gem/blob/master/lib/github-pages/plugins.rb#L21-L42){:target="_blank"} ，沒在允許清單的上線之後也用不了。
- 可以跟 AI 說： `優先使用 Jekyll 原生、Jekyll Plugin、免費開源的專案實現功能。`
- 可以跟 AI 說： `網站設計務必要重視 SEO 結構與 RWD 操作體驗。`
- 可以跟 AI 說： `設計新的 Theme 替代掉現有的` 。
- 部署到 Github Pages 的 CI/CD YAML 也可以請 AI 寫，步驟他也會教你。
- **如果你有新的頁面設計想調整建議回到 Claude Design 請設計調整，調整完再重新下載放回目錄；不要直接請 Claude Code 做，他沒什麼美感。**

#### **我用到的套件或功能**

**Jekyll 套件:**
- jekyll 4\.3 \+ kramdown \(GFM\) \+ kramdown\-parser\-gfm
- rouge — code highlight
- jekyll\-feed — Atom feed \(/feed\.xml\)
- jekyll\-sitemap — /sitemap\.xml
- jekyll\-paginate\-v2 — 文章分頁功能
- jekyll\-archives — 歸檔頁面
- jekyll\-seo\-tag — SEO Meta Tag / OG / Twitter Card meta
- jekyll\-redirect\-from — 短網址 redirect


**CSS / 前端資源:**
- GLightbox v3（CDN, MIT） — 圖片點擊放大 lightbox
- SCSS：assets/css/main\.scss
- [Font Awesome](https://fontawesome.com/){:target="_blank"} icon svg


**JavaScript 功能:**
- 抽屜選單（topbar menu / overlay / ESC 關閉）
- LQIP hydrator & Lazy Load — 文章圖片 Placeholder & 延遲載入
- Reading progress bar
- Client\-side TOC
- Search：/search\.json build\-time \+ 純 JS substring filter（無 lunr）


**第三方服務:**
- giscus — 留言（GitHub Discussions）

### 背景

2018 年起在 Medium 撰寫文章紀錄程式與生活，後來自己開發了一個小工具 [ZMediumToMarkdown](https://github.com/ZhgChgLi/ZMediumToMarkdown){:target="_blank"} 可以把 Medium 文章下載轉換成 Markdown 檔案備份，並順帶用 Jekyll \+ Chirpy theme 建立了獨立的 Blog 網站，部署在免費的 GitHub Pages 之上。


[![](https://repository-images.githubusercontent.com/65252/f2b7c780-70b6-11e9-85d2-f4bda8708a2d)](https://github.com/jekyll/jekyll){:target="_blank"}


Jekyll 是一套很方便的靜態網站產生工具，可以把 Markdown、模板轉換成 HTML，再把這些 HTML 檔案放到託管的服務，內容就能上線了。
- 我也用 Jekyll 做了一個自己的 Linktree: [https://link\.zhgchg\.li](https://link.zhgchg.li){:target="_blank"}
- `jekyll new blog` 就能快速 new 一個預設的 Blog 專案


**Jekyll 預設的 Blog 專案樣式非常陽春:**


![](/assets/6bf79c5b4dab/1*SB1pZSaMsMvUG8AiUla62w.png)


但是要找到滿意的 Jekyll Blog Theme 不太容易，有的是首頁漂亮、有的是文章內容頁體驗好、有的是列表頁不錯，就算是付費主題也很難找到完全滿意的；也有可能裝了都設定好了才發現少了什麼功能或是頁面，又只能捨棄再找，很痛苦。


[![](https://repository-images.githubusercontent.com/165360641/257ebaa6-61dd-40a4-b075-d21b546026d6)](https://github.com/cotes2020/jekyll-theme-chirpy){:target="_blank"}


**來來回回找好幾輪最終找到一個佈局不會太醜、支援 RWD、Blog 功能完善的主題 — [Theme Chirpy](https://github.com/cotes2020/jekyll-theme-chirpy){:target="_blank"} :**


![[Live Demo](https://chirpy.cotes.page/posts/text-and-typography/){:target="_blank"}](/assets/6bf79c5b4dab/1*Ayc3pF9qbahE5U2BnzixaQ.png)
