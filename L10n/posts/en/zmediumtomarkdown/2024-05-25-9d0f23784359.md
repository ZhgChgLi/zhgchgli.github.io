### Background

#### Asana

At my previous company Pinkoi, I first experienced the power of the Asana project management tool. Whether for internal project management or cross-team collaboration, Asana plays a key role in decoupling dependencies between people and tasks, improving collaboration efficiency.

In my previous company, all teams—from product to operations and business teams (such as HRBP, Finance, Marketing, BD…)—had a public team Project that served as a single collaboration entry point across teams. When other teams needed assistance, they could directly create a Task (or use a Template Task) in that Project (usually there was a "Need Help!" Section). The receiving team would then take over the Task and arrange accordingly.

Cross-team collaboration with the management team, such as procurement and recruitment processes, can be done by directly creating Tasks and tracking progress through them; collaboration with the business team includes marketing campaign planning and tasks requiring engineering assistance for modifications, etc.

**If you don't have Asana or similar project management tools:**

- For everything, we need direct communication with the other team. Direct communication is most effective for P0 issues, but in daily work, 90% of the time the issues are not P0. Direct person-to-person communication for all matters, big or small, is very inefficient and can [interrupt the other person's workflow](../11f6c8568154/).

- Task execution lacks transparency, and only the parties involved know the progress. For tasks involving multiple people, progress can only be confirmed through repeated communication. Additionally, supervisors find it difficult to grasp everyone's tasks for proper scheduling.

- Task scheduling: In daily work, we receive many tasks with varying priorities and directions. Having a tool helps collect and categorize similar issues, so they can be addressed together later. It also makes it easier to identify important tasks in daily planning.

- Task handover is facilitated by tools that record task details and execution status. When assistance from others is needed, the task can be quickly taken over based on the content in the Task.

Back to project management, Asana offers flexible, multi-dimensional, and automated project management tools that can be combined as needed.

#### Back to Reality

As mentioned earlier, the above describes the scenario when Pinkoi used the Asana project management tool; returning to an environment without any project management tools these past few months has made me even more aware of how important such tools are for work efficiency.

Currently, there is no modern project management tool available. Due to procurement constraints (need to control expenses), internal control issues (purely intranet), and personal data audit restrictions (must be on-premises), Asana cannot be directly implemented.

#### Framework

Plane.so is different from Asana's multi-dimensional flexibility but is composed of the following framework for project management:

- Issues: Similar to Asana Tasks, all work is created as Issues to be scheduled or kept as records.

![](/assets/9d0f23784359/1*pIAY63Q2gldFKfGJSce2qQ.png)

- Cycles: Similar to Sprints, a cycle is a time iteration or version. Each Issue can only belong to one Cycle.

![](/assets/9d0f23784359/1*FWPbQUefbahYxaTiIlpawA.png)

- Modules: Project, module, and categorization features. Each issue can be assigned to multiple modules.

![](/assets/9d0f23784359/1*R5tZxXT5Tu0WgMfy-xcxnQ.png)

- Layouts & Views: You can view Issues using Gantt charts, calendars, boards, lists, and Sheet modes. You can also save filter criteria and display settings as Views for quick access.

![](/assets/9d0f23784359/1*R5tZxXT5Tu0WgMfy-xcxnQ.png)

- Inbox: The Issue Proposed process allows creating proposal Issues. Only approved proposals will become actual Issues in the project; rejected ones are discarded.

![](/assets/9d0f23784359/1*pYH9rsIMI34wBIBv0hxfXg.png)

- Pages: A simple document feature to record work and product-related notes.

![](/assets/9d0f23784359/1*_8OiZDYMGTYrEfj_eRypfw.png)

- Drive: Similar to Google Drive team file features.

![](/assets/9d0f23784359/1*uEMf9QnbPmbVJ5fuq-NFmw.png)

Currently, this feature is not available in both the free version and the CE (Self-Hosted) version.

### Plane.so x Scrum Workflow Example

#### Architecture

![](/assets/9d0f23784359/1*Id2hjfltcwswSpYzG12Diw.png)

- Each team has its own Workspace

- Each team will have a main product Project.

- Projects: Other projects can be created, such as marketing campaigns, customer support projects, or projects for external collaboration, separate from the main product development project.

- Modules: Create Function Modules (Design, Frontend, Backend, App) to help Team Leads track progress and set OKRs or project goals as Modules (Increase Conversion Rate, OKR-1 Increase GMV…)

- Cycle: Create a Cycle based on the Sprint cycle. For example, if there is one Sprint per week, you can create `W12` or use the Monday date like `2024–05–27`.

- Since automatic Cycle creation is not available, you need to create future Cycles monthly or weekly in advance.

- Create an Issue for Every Task

- If possible, add Start Date & Due Date, Modules, and Priority to the Issue.

- **If an Issue keeps switching between In-Progress and Cycle (not completed in one Cycle), consider breaking the Issue into smaller parts for easier project management**

#### Workflow

- **Sprint Cycle: One Week**

- Backlog: Create an Issue for any task or idea, set State = Backlog, **fill in Estimate and Priority**

- **Weekly Sprint Planning Meeting: Select Issues from the Backlog and ongoing Issues (To Do or In Progress), set Priority/Estimate, plan the current Sprint, and add them to the Cycle**

- If there is an urgent Issue added during the Sprint, it should be directly created in the current week's Cycle.

- Daily Stand-up: Spend 15 minutes every morning quickly sharing the status of each Issue

- Change issue status to ToDo/In Progress when preparing to start or starting execution

- Change the Issue status to Done when completed, or add a Review State.

- **Weekly Friday Sprint Review Meeting: Simply review this week's Issues (not planning for next week), quickly go over completed Issues, and ensure Estimates are filled in for future statistics.**

- Try to ensure all Issues in the Cycle are completed by Friday each week. **Unfinished Issues should be added to the next week's Cycle or moved to Pending/Cancel**.

- Continuously iterate the above process to manage all Issues and Projects

### ⚠️⚠️Disclaimer⚠️⚠️

> The above is just a workflow example. Please note that there is no perfect process, only the one that fits your team. Use the framework provided by Plane.so to creatively find the best project management method.
