### Google Apps Script を使って無料で Github リポジトリのスター通知を3ステップで作成する

GASでGithub Webhookを連携し、スターのLike通知をLineに転送する方法

#### はじめに

オープンソースプロジェクトのメンテナとして、お金や名声のためではなく、ただ一つの **虚栄心** のために；新しい⭐️スターを見かけるたびに、心の中で密かに喜ぶ；時間と労力をかけて作ったプロジェクトが、本当に誰かに使われていて、同じ問題を抱える友人の役に立っていると実感できる。

![[Star History Chart](https://star-history.com/#ZhgChgLi/ZMarkupParser&Date){:target="_blank"}](/assets/382218e15697/1*fUtW942huwnSbPF-ar4OCQ.png)

[Star History Chart](https://star-history.com/#ZhgChgLi/ZMarkupParser&Date){:target="_blank"}

そのため、⭐️スターの観察には少し強迫観念があり、時々Githubをリロードして⭐️スターの数が増えているか確認していました。そこで、誰かが⭐️スターを押したときに自動で通知が届くような、もっと能動的な方法はないかと考えました。手動で追跡・確認する必要がありません。

#### 既存のツール

まず既存のツールを探すことを考え、[Github Marketplace](https://github.com/marketplace?type=apps&query=Star+Notifier+){:target="_blank"}で検索してみました。いくつか優れたツールがありました。

![](/assets/382218e15697/1*wQZ6-F77v2SEepm90YAF-A.png)

いくつか試しましたが、期待通りの効果が得られませんでした。すでに動作しなくなっているものもあり、5/10/20個の⭐️星がついたときだけ通知が送信されるものもあります（私は小規模なので、⭐️が1つ増えるだけでとても嬉しいです😝）。通知はメールのみで、SNSでの通知が欲しいです。

さらに、「見栄」のためだけにアプリを作るのは心配で、セキュリティリスクがあるのではないかと不安です。

iOS 上の [Github](https://apps.apple.com/tw/app/github/id1477376905){:target="_blank"} アプリや [GitTrends](https://github.com/brminnick/GitTrends){:target="_blank"} などのサードパーティ製アプリもこの機能をサポートしていません。

### 自分で作る Github Repo Star Notifier

以上を踏まえ、実は Google Apps Script を使って無料かつ迅速に自分の Github リポジトリのスター通知システムを作成できます。

### 2024/10/12 更新

> *⚠️⚠️⚠️*

> *Line Notifyは[2025/04/01に終了します](https://notify-bot.line.me/closing-announce){:target="_blank"}ので、**私の最新記事「[10分でできるLine NotifyからTelegram Bot通知への移行](../6922e90ba90c/)」**を参考に、Telegramでの通知連携に切り替えてください。*

#### 準備作業

本文では通知手段として Line を使用していますが、他のメッセージアプリで通知したい場合は [ChatGPT](https://chat.openai.com){:target="_blank"} に実装方法を問い合わせてください。

![[ChatGPT](https://chat.openai.com){:target="_blank"} に Line Notify の実装方法を質問](/assets/382218e15697/1*NzEyi3zdzD5QDhLvpsFocA.png)

[ChatGPT](https://chat.openai.com){:target="_blank"} に Line Notify の実装方法を尋ねる

`lineToken` ：

- [Line Notify](https://notify-bot.line.me/my/){:target="_blank"} にアクセスしてください。

- Lineアカウントにログインした後、画面を下にスクロールして「Generate access token (For developers)」セクションを見つけてください。

![](/assets/382218e15697/1*GA_ORi8TX3N8jPSxX4OqHw.png)

- 「Generate token」をクリックしてください

![](/assets/382218e15697/1*qu1mFEhu8f6_bXRvW0uFpw.png)

- Token Name：メッセージの前に表示されるロボットのタイトル名を入力してください（例：`Github Repo Notifer: XXXX`）

- メッセージの送信先を選択：私は `1-on-1 chat with LINE Notify` を選び、LINE Notify公式ボットを通じて自分にメッセージを送信します。

- 「Generate token」をクリックしてください

![](/assets/382218e15697/1*DMXhPQBiBQH_dTYA4mayHw.png)

- 「コピー」を選択してください

- **そしてトークンを控えておいてください。後で忘れた場合は再生成が必要で、再度確認することはできません。**

`githubWebhookSecret` ：

- [Random.org](https://www.random.org/strings/?num=1&len=32&digits=on&upperalpha=on&loweralpha=on&unique=on&format=html&rnd=new){:target="_blank"} にアクセスしてランダムな文字列を生成してください。

![](/assets/382218e15697/1*eh6baff8FBN7e_m8YjSy-Q.png)

- コピーしてこのランダムな文字列をメモしてください

私たちはこの文字列を Github Webhook と Google Apps Script 間のリクエスト認証の媒介として使用します。

> [GASの制限](https://issuetracker.google.com/issues/67764685?pli=1){:target="_blank"} により、`doPost(e)` 内で `Headers` の内容を取得できないため、Github Webhookの[標準的な認証方法](https://docs.github.com/en/webhooks-and-events/webhooks/securing-your-webhooks){:target="_blank"} を使用できず、`?secret=` クエリによる文字列マッチ認証を手動で行う必要があります。

#### Google Apps Script の作成

**[Google Apps Script](https://script.google.com/home/start){:target="_blank"}** にアクセスし、左上の「+ 新しいプロジェクト」をクリックします。

![[**Google Apps Script**](https://script.google.com/home/start){:target="_blank"}](/assets/382218e15697/1*ajoOp3ZLc88ecEtYbUVP4A.png)

[**Google Apps Script**](https://script.google.com/home/start){:target="_blank"}

左上の「名前のないプロジェクト」をクリックして、プロジェクト名を変更します。

![](/assets/382218e15697/1*Gv2KLHa-7qNnL71_jBblGA.png)

ここではプロジェクト名を `My-Github-Repo-Notifier` として、今後の識別を容易にしています。

**コード入力エリア：**

```javascript
// 定数
const lineToken = 'XXXX';
// 自分のLine Notifyボットトークンを生成：https://notify-bot.line.me/my/
const githubWebhookSecret = "XXXXX";
// シークレット文字列を生成：https://www.random.org/strings/?num=1&len=32&digits=on&upperalpha=on&loweralpha=on&unique=on&format=html&rnd=new

// HTTP Get/Post ハンドラー
// Getメソッドは許可しない
function doGet(e) {
  return HtmlService.createHtmlOutput("Access Denied!");
}

// Github WebhookはPostメソッドで送信される
function doPost(e) {
  const content = JSON.parse(e.postData.contents);
  
  // セキュリティチェック、リクエストがGithub Webhookからのものか確認
  if (verifyGitHubWebhook(e) == false) {
    return HtmlService.createHtmlOutput("Access Denied!");
  }

  // starイベントのpayload content["action"] == "started"
  if(content["action"] != "started") {
    return HtmlService.createHtmlOutput("OK!");
  }

  // メッセージ作成
  const message = makeMessageString(content);
  
  // メッセージ送信、SlackやTelegramにも変更可能
  sendLineNotifyMessage(message);

  return HtmlService.createHtmlOutput("OK!");
}

// メソッド
// メッセージ内容を作成
function makeMessageString(content) {
  const repository = content["repository"];
  const repositoryName = repository["name"];
  const repositoryURL = repository["svn_url"];
  const starsCount = repository["stargazers_count"];
  const forksCount = repository["forks_count"];

  const starrer = content["sender"]["login"];

  var message = "🎉🎉「"+starrer+"」があなたの「"+repositoryName+"」リポジトリにスターを付けました 🎉🎉\n";
  message += "現在のスター数: "+starsCount+"\n";
  message += "現在のフォーク数: "+forksCount+"\n";
  message += repositoryURL;

  return message;
}

// リクエストがGithub Webhookからのものか検証
// GASの制限により (https://issuetracker.google.com/issues/67764685?pli=1)
// ヘッダーの内容を取得できない
// そのためGithub Webhookの標準検証方法は使えない (https://docs.github.com/en/webhooks-and-events/webhooks/securing-your-webhooks)
// ?secret=XXX の手動マッチング検証のみ可能
function verifyGitHubWebhook(e) {
  if (e.parameter["secret"] === githubWebhookSecret) {
    return true
  } else {
    return false
  }
}

// -- メッセージ送信 --
// Line
// 他の送信方法はChatGPTに聞いてください
function sendLineNotifyMessage(message) {
  var url = 'https://notify-api.line.me/api/notify';
  
  var options = {
    method: 'post',
    headers: {
      'Authorization': 'Bearer '+lineToken
    },
    payload: {
      'message': message
    }
  }; 
  UrlFetchApp.fetch(url, options);
}
```

`lineToken` と `githubWebhookSecret` に前のステップでコピーした値を設定してください。

![](/assets/382218e15697/1*ZBj7EvEXfn0nuRgfotI6IA.png)

![](/assets/382218e15697/1*YeOPSIKo6x6f-Qa3ymeT0Q.png)

![](/assets/382218e15697/1*p7QiiYISmberMJPGQINr1w.png)

![](/assets/382218e15697/1*kYmtS0WBI-NRXxDuUTrU8A.png)

#### Github Webhook 設定

- [Github](https://github.com/){:target="_blank"} に戻る

- Organizations（その中のすべてのリポジトリ）または単一のリポジトリに対してWebhookを設定し、新しい⭐️スターを監視できます。

Organizations / Repo に入り、「Settings」-> 左側の「Webhooks」を見つけ、「Add webhook」をクリック：

![](/assets/382218e15697/1*Ke5ZarGC8ODrFLj8LsBNFg.png)

![](/assets/382218e15697/1*PUHcpcJkbL4d7xTI5A99PA.png)

- Payload URL **：** `GASのURL` を入力し、URLの後ろに自分のセキュリティ検証文字列 `?secret=githubWebhookSecret` を手動で追加します。  
  例えば、あなたの `GASのURL` が `https://script.google.com/macros/s/XXX/exec` で、`githubWebhookSecret` が `123456` の場合、**URLは `https://script.google.com/macros/s/XXX/exec?secret=123456` となります。**

- **Content type：** `application/json` を選択してください

- **どのイベントでこのウェブフックをトリガーしますか？**  
  **「** `Let me select individual events.` **」を選択してください。**  
  **⚠️「** `Pushes` **」のチェックを外してください。**  
  **️️️️⚠️「 `Watches` 」にチェックを入れてください。なお、「 `Stars` 」ではありません（ただし、Starsも星をクリックする状態を監視するので、Starsを使う場合はGASのアクション判定も調整が必要です）。**

- 「 `Active` 」を選択してください。

- 「Add webhook」をクリックしてください

- 設定完了

設定した Organizations のリポジトリ / リポジトリで「Star」をクリックするか、一度アンスターしてから再度「Star」をクリックしてください：
