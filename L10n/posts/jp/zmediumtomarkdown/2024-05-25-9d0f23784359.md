### Plane.so 無料でオープンソース、セルフホスト対応のAsana/Jira類プロジェクト管理ツール

Plane.so プロジェクト管理ツールの使い方と Scrum プロセスとの連携方法

![](/assets/9d0f23784359/1*BQgytUYomqHcD33LWA-ZaQ.png)

### ️⚠️️️️⚠️️️️⚠️️️️2025年アップデート️ ⚠️️️️⚠️️️️⚠️️️️

Plane.so の使用は推奨されません。理由は、Plane.so は当初オープンソースプロジェクトとセルフホスティングのサポートで成長しましたが、後のバージョンでは多くの機能が有料プランに移行されました。セルフホスティングでも機能を有効化するには証明書の購入が必要で、ユーザー数にも制限があります。これでは当初の期待に沿っていません。また、料金プランも非常に複雑で、終身プラン、Pro、Business などがあり、たとえお金を払っても新しいプランが出て新機能がそちらだけに追加される可能性があります。

### 背景

#### Asana

前職のPinkoiで初めてAsanaの強力さを実感しました。社内プロジェクト管理やチーム間の協力において、Asanaは人と人、人とタスクの依存関係をうまく調整し、協力の効率を高める役割を果たします。

前職では、プロダクトチームから経営やビジネスチーム（例：HRBP、Finance、Marketing、BD…）まで、すべてのチームにチーム公開のプロジェクトがあり、チーム間の単一の協力入口として機能していました。ほかのチームが支援を必要とする場合、そのプロジェクト内に直接タスクを作成（テンプレートタスクからも可能）し、通常「Need Help!」セクションがあります。チーム内でタスクを受け取った後、対応を進めていました。

経営チームとのクロスチーム協力では、購買や採用プロセスなどのタスクを直接作成し、それを通じて進捗を追跡できます。ビジネスチームとの協力では、マーケティング活動の計画やエンジニアリングの支援が必要なタスクなどがあります。

**Asanaや類似のプロジェクト管理ツールがない場合：**

- あらゆることにおいて、相手チームと直接コミュニケーションを取る必要があります。P0の案件は直接のやり取りが最も効果的ですが、日常的には90%がP0ではありません。大小問わずすべてを直接人対人で話すのは非効率であり、相手の作業フローを[妨げてしまいます](https://medium.com/pinkoi-engineering/2021-pinkoi-tech-career-talk-%E9%AB%98%E6%95%88%E7%8E%87%E5%B7%A5%E7%A8%8B%E5%9C%98%E9%9A%8A%E5%A4%A7%E8%A7%A3%E5%AF%86-11f6c8568154){:target="_blank"}。

- タスクの進行状況が不透明で、関係者同士だけが状況を把握しています。複数の関係者がいる場合は、コミュニケーションを通じて何度も進捗を確認するしかありません。また、上司もメンバーのタスク状況を把握しづらく、適切な割り振りが難しいです。

- タスクの割り当てについて、日常的に多くのタスクを受け取ります。優先度は高いものも低いものもあり、方向性も多岐にわたります。ツールがあって初めて、同じ種類の問題を収集・分類でき、後でまとめて解決することが可能です。日常の作業管理も重要なタスクを見つけやすくなります。

- タスクの引き継ぎでは、ツールがタスクの詳細や進行状況を記録します。もし他の人の協力が必要な場合でも、タスク上の内容から素早く引き継ぐことができます。

プロジェクト管理に戻ると、Asana は柔軟で多次元的かつ自動化されたプロジェクト管理ツールを提供しており、ニーズに応じて自由に組み合わせて使用できます。

> Asana の使い方は多様ですが、以下にいくつかの使用例を挙げます。まずはニーズを明確にしてから、関連する Asana のサンプルを適用することをおすすめします。

> Asana 台湾の代理店も完全な教育トレーニングを提供しています。興味があれば、彼らに連絡してください。

> （本記事には広告を含みません）

チームのメインプロジェクトとは別に、Scrumプロジェクトを作成して管理します（Asanaのタスクは複数のプロジェクトに同時に追加可能）。各スプリントのタスク実行内容を振り返ります。

#### 現実に戻る

前述の通り、前職のPinkoiではAsanaのプロジェクト管理ツールを使用していましたが、この数ヶ月間、プロジェクト管理ツールのない環境に戻ってみて、ツールが仕事の効率にどれほど重要かを改めて実感しました。

現在の環境には最新のプロジェクト管理ツールがなく、調達（支出管理が必要）、内部統制問題（完全なイントラネット）、個人情報監査の制限（ローカル環境必須）により、Asana を直接導入できません。

以上の環境制限を踏まえ、まずはオープンソースかつセルフホスト対応のプロジェクト管理ツールに絞って検討しました。見つかった解決策はRedmine、OpenProject、Taigaなどでしたが、いくつか試した結果、機能不足やUI/UXの使い勝手の悪さで期待外れでした。そんな中、偶然2023年1月に新しくリリースされたPlane.soというプロジェクト管理ツールを見つけました。

### Plan.so 紹介

#### 簡介

[![](https://plane.so/opengraph-image-4usi79.png?opengraph-image.0b2l1f2243yp~.png)](https://plane.so/){:target="_blank"}

Plane は 2022 年に設立された、アメリカ・デラウェア州とインドに拠点を置くスタートアップ企業です。現在、[Linkedin](https://www.linkedin.com/company/planepowers/){:target="_blank"} と [Github](https://github.com/makeplane/plane){:target="_blank"} を見ると、開発者の大部分がインドにいることが分かります。同社は現在、OSS Capital からの出資を受けて400万ドルのシード資金を調達しています。

[![](https://plane.so/api/media/file/Announcing-git-hub-stars-1200x630.webp)](https://plane.so/blog/how-we-got-to-20k-github-stars){:target="_blank"}

現在 Plane は Github の[プロジェクト管理カテゴリで第1位](https://github.com/topics/project-management){:target="_blank"}を占めており、[AGPL-3.0 ライセンス](https://github.com/makeplane/plane#AGPL-3.0-1-ov-file){:target="_blank"}でオープンソース化されています。初版は2023年1月にリリースされ、現在も開発段階で正式なリリース版はまだ提供されていません。

> **ご注意：** ⚠️ **オープンソースは無料ではありません** ⚠️ **。** GithubやGitlabのように、Asana、Jira、Clickupなど多くのGithubに似たプロジェクト管理ツールがありますが、Gitlabのように十分に使いやすく競合できるオープンソース製品はまだありません。Planeの目標は、プロジェクト管理ツールのGitlabになることです。

- 約2〜3週間ごとに開発版を更新しており、一部の調整には大きな差異があったり、まだセキュリティ問題が残っている場合があります。

#### フレームワーク

Plane.so は Asana の多次元で柔軟な機能とは異なり、以下のフレームワークで構成されており、プロジェクト管理に使用されます：

- Issues：Asanaのタスクと同様に、すべての作業をIssueとして作成し、担当者を割り当てたり記録として残したりします。

![](/assets/9d0f23784359/1*pIAY63Q2gldFKfGJSce2qQ.png)

- Cycles：スプリントのようなもので、1つのイテレーション期間やバージョンを指します。各Issueは1つのCycleにのみ存在できます。

![](/assets/9d0f23784359/1*FWPbQUefbahYxaTiIlpawA.png)

- Modules：プロジェクト、モジュール、カテゴリ機能で、各Issueに複数のModulesを追加可能。

![](/assets/9d0f23784359/1*R5tZxXT5Tu0WgMfy-xcxnQ.png)

- Layouts & Views：ガントチャート、カレンダー、カンバン、リスト、シートモードで Issues を表示できます。また、フィルター条件や表示方法を View として保存し、素早く確認することも可能です。

![](/assets/9d0f23784359/1*R5tZxXT5Tu0WgMfy-xcxnQ.png)

- Inbox：Issue Proposed フローでは、提案された Issue を作成できます。承認されて初めて実際にプロジェクト内の Issue として作成され、承認されなければ直接却下されます。

![](/assets/9d0f23784359/1*pYH9rsIMI34wBIBv0hxfXg.png)

- Pages：簡単なドキュメント機能で、作業や製品の内容を記録できます。

![](/assets/9d0f23784359/1*_8OiZDYMGTYrEfj_eRypfw.png)

- Drive：Google Driveのチームファイル機能に似ています。

![](/assets/9d0f23784359/1*uEMf9QnbPmbVJ5fuq-NFmw.png)

現在のところ、無料版とCE（セルフホスト）版にはこの機能はありません。

私たちは迅速かつ無料で、直接 Plane Cloud バージョンを使い始めることができます：

[![](https://app.plane.so/assets/og-image-DL-1GVBU.png)](https://app.plane.so/){:target="_blank"}

#### ワークスペース

![](/assets/9d0f23784359/1*yxtGND6xaZk8ey3vuvqMiw.png)

- 初めて Plane.so にアクセスすると、最初の Workspace を作成する必要があります。

- Workspace は Asana のワークスペースと同様で、1つのアカウントで複数の Workspace に参加できます。

- 小規模な会社でチーム間で使用する場合は、同じ Workspace 内で利用可能です。

- 大企業の複数チームの場合、Plane には Asana のような Team 機能や Project のグループ分け機能がありません。同じ Workspace 内だと Projects が混乱するため、チームごとに Workspace を分けることをおすすめします。

**作成後は Workspace のドロップダウンメニューから別の Workspace に切り替えられ、ここから Workspace Settings にもアクセスできます：**

![](/assets/9d0f23784359/1*uKD3DSeeZdEzS3meTqGhiw.png)

**設定がある：**

- General Workspace のプロフィール画像、名前、URL

- Billing and plans 料金とプラン

- Integrations サードパーティ連携は、現在無料版で Github と Slack のみ対応しています。

- Imports インポート機能は現在、Jira と Github Project のインポートのみ対応しています。

- Exports エクスポート機能は、現在 csv、excel、json 形式のエクスポートのみ対応しています。

- Webhooks API トークン、自分で API を連携する

**重要な設定は Members で、チームメンバーを Workspace に招待する必要があります:**

![](/assets/9d0f23784359/1*as1_IkYpiBHu7uB6ftuZIw.png)

![](/assets/9d0f23784359/1*xPF2Bdp8KywAOR4AoyXWbA.png)

身分別：

- Guest/Viewer は現在の機能上、大きな違いはなく、Issue、Comment、Emoji の閲覧のみ可能です。異なる組織のメールアドレスからの外部ユーザーは Guest として扱われ、同じ組織内のユーザーは Viewer として扱われます。

- メンバーはすべての機能を操作できます

- 管理者は設定にアクセスできます

#### Home ホームページ

![](/assets/9d0f23784359/1*dy4oF2haCVir8iU5OYGq3A.png)

- Home ワークスペース全体のすべてのプロジェクトとメンバーの状態

- Analytics 全メンバーおよび Issues の分析

- Projects すべてのプロジェクト

- All Issues すべてのプロジェクトの Issues

- Active Cycles すべてのプロジェクトの現在の Cycle 状態

- Notifications Issues の通知

#### プロジェクト

Projects に入り、すべての公開および参加している Projects を確認します:

![](/assets/9d0f23784359/1*F3721uSBCQZYQjUAPNELJw.png)

- プロジェクト名、概要、カバー画像、プレフィックス（Issue Alias 例: `APP-1`）

- プロジェクト権限：Public ワークスペースに参加している全員が閲覧・参加可能；Private 招待されたメンバーのみ参加可能

- リード：プロジェクトの主要担当者

**プロジェクト右上の「…」でできること：**

![](/assets/9d0f23784359/1*KSNKPmXhVBE4odSI9gvzwg.png)

- お気に入りに追加し、Pinを「マイフェイバリット」（Your Projectsの上）に固定

- Publish は公式の [Roadmap Project](https://sites.plane.so/plane/459cfcdf-dc9c-49a6-9d4b-a89f91440189/?board=kanban){:target="_blank"} のように、外部公開用のリンクを生成します。

- Draft Issues 保存された下書きの Issues を確認する

- Archives 封印された Issues を表示する

**その他の設定：**

![](/assets/9d0f23784359/1*vlv6RvQu1LBTaHSWarnecg.png)

- General：プロジェクトの一般設定

- Members：プロジェクトメンバーと権限管理

- States：プロジェクトのIssueステータス（後述します）

- Labels：プロジェクトのラベル管理

- Features：有効にする機能を選択可能（デフォルトではInbox機能は無効）

- Estimates：プロジェクトの見積もり時間設定（後述します）

- Integrations：サードパーティ連携（Workspaceで事前に有効化が必要）

- Automations：現在の無料版では、一定時間を超えたClosed Issuesの自動アーカイブと、一定時間を超えた未完了Issuesの自動クローズのみ対応しています。

#### **Issues**

![](/assets/9d0f23784359/1*0LbfR7I6buTOg4tHc9dmUw.png)

![](/assets/9d0f23784359/1*UbcyuxuqrK7j67B-AJJ1gQ.png)

![](/assets/9d0f23784359/1*Y95JgIrXhy5qYjtL8c2YAg.png)

- 入ると左側の Projects からプロジェクトを作成できます。

- Asana と異なり、Plane の Issue は一つの Project のみ追加可能です。

- 右上の角で表示モードを切り替え可能です。

- デフォルトではすべてのサブイシューが展開されています。見づらい場合は、Display -> Show sub-Issues のチェックを外してください。

**「Create Issue」をクリックしてIssueを作成開始：**

![](/assets/9d0f23784359/1*2tDzhxizocI2G8xAaC_Tqw.png)

![](/assets/9d0f23784359/1*FBBTuudbXscDsT7ltHng9A.png)

- ドラフトとして保存可能な Issues

- 文字スタイルやコードブロックのサポート

- Markdown対応

- テキストの回り込み対応、画像を直接ドラッグ＆ドロップでアップロード可能

- 複数担当者のサポート（Asanaより便利で、Asanaは1タスクにつき1人の担当者のみ）

- Priorityを選択すると、異なるPriorityに応じて異なるハイライトスタイルが適用されます（現在、Priorityのカスタマイズは不可）

![](/assets/9d0f23784359/1*0EXpzC9Xq2lKvsHOPWdDbA.png)

- Modules を選択すると、複数の Modules を追加できます。例えば：`ログイン最適化`、`App` など（設定は後で紹介します）

- Cycle を選択すると、どのスプリントで行うかを一つだけ選べます。例えば：`W22`、`S22`、`2024–05` …（設定は後で説明します）

- 現在はカスタム Issue プロパティをサポートしていません

- 「Add parent」を選択し、この Issue を Sub-Issue として Parent Issue に追加します。

- Labels の選択（別名 Tag 機能）

- Start Date、Due Date を選択します…（現時点では正確な時間の指定や繰り返し Issue はサポートしていません）

### Plane.so x Scrum ワークフロースタイル例

#### アーキテクチャ

![](/assets/9d0f23784359/1*Id2hjfltcwswSpYzG12Diw.png)

- 各チームにはそれぞれのワークスペースがあります。

- 各チームには主要な製品プロジェクトがあります。

- Projects：マーケティング広告プロジェクト、カスタマーサポートプロジェクト、外部協力プロジェクトなど、製品の主要開発プロジェクトとは別に作成可能です。

- Modules：Function Modules（設計、フロントエンド、バックエンド、アプリ）を作成し、チームリードが追跡しやすくします。OKRやプロジェクト目標をModuleとして設定します（コンバージョン率向上、OKR-1 GMV向上など）。

- Cycle：Sprintのサイクルに合わせてCycleを作成します。例えば、毎週1つのSprintの場合は `W12` または月曜日の日付に合わせて `2024–05–27` を作成できます。

- 現在はCycleを自動作成できないため、毎月または毎週、あらかじめ将来のCycleを作成する必要があります。

- どんな仕事でもIssueを作成する必要があります。

- Issueには可能であればStart Date & Due Date、Modules、Priorityを追加してください。

- **もしIssueがずっとIn-ProgressとCycleを行き来している場合（1つのCycleで完了しない場合）、Issueを細分化してプロジェクト管理をしやすくすることを検討してください。**

#### フロー

- **スプリント期間：1週間**

- Backlog：やるべき作業やアイデアはすべてIssueとして登録し、StateをBacklogに設定し、**EstimateとPriorityを記入**

- **毎週月曜日のスプリントプランニングミーティング：バックログからIssueを選び、進行中のIssue（To Do または In Progress）を確認し、優先度や見積もりを設定して、今回のスプリントに組み込み、サイクルに追加する**

- スプリント中に割り込みで実行するIssueがある場合も、必ず今週のCycleに直接追加する必要があります。

- Daily Stand-up：毎朝15分間で各自のIssueの進捗状況を手短に共有する

- Issueを実行準備中、実行開始に変更し、状態をTo Do/In Progressにする

- Issueの状態を完了（Done）に変更するか、または新たにReviewステートを作成する

- **毎週金曜日のスプリントレビュー会議：今週のIssueを単純にレビュー（翌週の計画ではありません）。完了したIssueを素早く振り返り、後で集計しやすいようにEstimateを必ず記入します。**

- 毎週金曜日には、できるだけCycle内のすべてのIssueが完了していることを確認し、**未完了のIssueは次週のCycleに追加するか、Pending/Cancelに変更してください**。

- 上記のプロセスを継続的に繰り返し、すべてのIssuesとProjectsを管理する

### ⚠️⚠️声明⚠️⚠️

> 以上はあくまでワークフローの例です。完璧なプロセスは存在せず、チームに合った方法が最適です。Plane.so が提供する構造を参考に、創意工夫して最適なプロジェクト管理方法を見つけてください。

#### API

Plane.so は非常にクリーンなフロントエンドとバックエンドの分離アーキテクチャを採用しているため、充実した API が利用できます。Workspace Settings で API Tokens を作成し、API リクエストヘッダーに `X-API-Key` を付けることで利用可能です。API エンドポイントのリクエスト方法は公式の [API ドキュメント](https://docs.plane.so/api-reference/introduction){:target="_blank"} を参照してください。

> しかし、公式ドキュメントはまだ不完全で、多くのリクエスト方法が記載されていません。最も早い方法はブラウザの開発者ツールでネットワークリクエストを確認し、公式がどのようにAPIを呼び出しているかを見て、自分のキーを使うことです。

#### Issue コメントで中国語入力時、文字選択後すぐに送信される問題

公式に Issue を開いてソースコードを確認しましたが、修正される可能性は低そうです。最初から選択言語が必要だとは考慮されておらず、キーボードの Enter イベントで直接コメントを送信する仕様になっています。

**ブラウザ拡張機能の回避策：**

こちらは自作の Workaround JavaScript スクリプトで、Enter イベントをフックするものです。

1.まずは JavaScript ブラウザ注入プラグインをインストールします：

[![](https://lh3.googleusercontent.com/1TSw8ZPwr2VvUZlY1cbAPOWo5cisrfpazmTpfGH4YWGTLCJLUq23LMSKuS5ee-bBP5C0GL8wsbpV9iQ9zLEQ6Dv8Vw=s128-rj-sc0x00ffffff)](https://chromewebstore.google.com/detail/user-javascript-and-css/nbhcbdghjpllgmfilhnhkllmkecfmpld){:target="_blank"}

以上は Chromium 共通の拡張機能であり、他のブラウザでも類似の JavaScript Inject 注入ツールを探すことができます。

2. Plane.so に戻り、拡張機能をクリック -> 「+」をクリックしてください

![](/assets/9d0f23784359/1*Ho5_6Qhd4_qU2Srhtoq7Qw.png)

3. Plane.so に以下の JavaScript を注入する

```javascript
document.addEventListener('keydown', function(event) {
 if (event.key === 'Enter' \\|\\| event.keyCode === 13) { // event.keyCode は古いブラウザ用
  const focusedElement = document.activeElement;
  const targetButtons = focusedElement.parentElement.parentElement.parentElement.parentElement.parentElement.querySelectorAll('button[type="submit"]');
   if (targetButtons.length > 0 && targetButtons[0].textContent.trim().toLowerCase() === "comment") {
    console.log("HIT");
    // アクティブな要素にフォーカスを当て、カーソルを末尾に置く
    focusedElement.focus();
    if (window.getSelection) {
     var range = document.createRange();
     var selection = window.getSelection();
     range.selectNodeContents(focusedElement);
     range.collapse(false);
     selection.removeAllRanges();
     selection.addRange(range);
    }
    event.stopImmediatePropagation();
   }
 }
},true);
```

![](/assets/9d0f23784359/1*eQqy00aFRtImZX-1imQJng.png)

- コードを貼り付けた後、「Save」を押して保存します。

Plane.so に戻り（リフレッシュ）、Issue を開いてコメント機能をテストします。

![](/assets/9d0f23784359/1*oqmdLydKifdqHT8GNIuyow.png)

- Enterキーでの選択は自動送信されなくなり、Space + Shift + Enterで改行ができます。コメントの送信は手動で「Comment」をクリックしてください。
